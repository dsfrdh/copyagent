# PRD: 热点选题文案生成一体化 & 内容形式定制

## 概述

将 CopyAgent 的「热点选题」「文案生成」「定时配置」三大模块整合为统一的「热点选题 & 文案生成」入口，置顶为一级导航；新增「内容形式」定制能力，让 AI 根据用户指定的表现形式（单人口播、剧情演绎、测评对比等）调整文案脚本结构。

## 背景

当前 CopyAgent 存在三个割裂的入口：

| 入口 | 位置 | 功能 |
|------|------|------|
| 热点选题 | 导航第 4 位 | 热点搜索 → 选题生成 → 痛点文案 |
| 文案生成 | 导航第 5 位 | 向导模式 / 高级模式（自由、仿写、改写、组合） |
| 定时配置 | 设置页内 | 每日定时生成时间 + 默认参数 |

用户痛点：
1. **路径分散** — 热点选题生成的文案想换个风格，需要切到「文案生成」重来
2. **配置隐藏** — 定时生成和默认参数藏在设置里，不易发现和调整
3. **形式单一** — 所有文案都是单人口播风格，无法生成剧情演绎、测评对比等形式

## 目标

1. **统一入口** — 热点选题置顶，文案生成 + 定时配置并入同一页面
2. **路径切换** — 通过「创作路径」radio 在热点驱动 / 向导模式 / 高级模式之间切换
3. **内容形式** — 新增 8 种内容形式，AI 根据选中的形式调整脚本结构
4. **配置就近** — 定时生成时间 + 默认参数与创作入口同页，调整方便

## 功能详述

### 1. 导航重组

```
(置顶) 🔥 热点选题 & 文案生成
        🏠 首页
        📚 知识库
        🔍 爆款拆解
        📋 历史记录
        🧠 偏好记忆
        ⚙️ 设置 (精简为 API 配置 + 系统信息)
```

### 2. 合并后的热点选题页

```
┌─────────────────────────────────┐
│ 🔥 热点选题 & 文案生成          │
├─────────────────────────────────┤
│ 创作路径: ○热点驱动 ○向导 ○高级 │
│ 🎬 内容形式: [单人口播 ▼]      │
│        单人口播 — 单人面对镜头…  │ ← 根据选项动态显示说明
├─────────────────────────────────┤
│ (根据创作路径条件渲染)          │
│                                 │
│  热点驱动:                      │
│    Step 1 → 产品 + 平台 + 时间   │
│    Step 2 → 热点分析结果        │
│    Step 3 → 选题选择            │
│    Step 4 → 文案生成 + 长度/风格 │
│                                 │
│  向导模式:                      │
│    Step 1-5 渐进式文案向导      │
│                                 │
│  高级模式:                      │
│    自由创作 / 仿写 / 改写 / 组合 │
│                                 │
├─────────────────────────────────┤
│ ▶ ⏰ 定时生成配置 (可折叠)      │
│ ▶ 📝 自动生成默认参数 (可折叠)   │
└─────────────────────────────────┘
```

### 3. 内容形式选项

| 选项 | AI 行为 | 适用场景 |
|------|---------|----------|
| **单人口播** | 单人面对镜头口播，句式简短自然 | 泛知识、口播号 |
| **剧情演绎** | 1-2 角色对话/情景剧，人物互动 | 带货短剧、情景号 |
| **测评对比** | 产品对比/前后对比，数据+体验 | 评测号、种草号 |
| **Vlog分享** | 第一人称生活分享，日常场景 | Vlog 号、生活号 |
| **干货讲解** | 知识科普+产品植入，先专业后转化 | 知识付费、IP 号 |
| **对话访谈** | 两人一问一答，角色自然 | 访谈号、双人号 |
| **开箱体验** | 收到→使用的完整过程，细节+感受 | 开箱号、种草号 |
| **场景种草** | 聚焦一个具体场景，渲染需求再解决 | 场景号 |

每种形式对应一段 **prompt instruction**，注入到所有生成模式的 LLM 请求中。默认值为「单人口播」，保持向后兼容。

### 4. 定时配置移动

| 原位置 | 新位置 |
|--------|--------|
| 设置 → ⏰ 定时生成 | 热点选题页底部 expander |
| 设置 → 📝 自动生成默认参数 | 热点选题页底部 expander |

自动生成默认参数新增 `auto_content_format`（默认内容形式），定时任务执行时会携带此参数。

### 5. Session State 设计

所有现有 key 保留（hotspot_*、wizard_*、imit_* 前缀天然隔离）。新增：

| Key | 默认值 | 用途 |
|-----|--------|------|
| `creation_path` | `"热点驱动"` | 当前选中的创作路径 |
| `content_format` | `"单人口播"` | 当前选中的内容形式 |

### 6. 爆款拆解跳转

「爆款拆解」页的「用这个结构仿写」按钮不再跳转到已删除的「文案生成」页，而是设置 `creation_path = "高级模式"` + `mode = "仿写爆款"`，用户切到热点选题页时自动定位到高级仿写模式。

## 技术变更

| 文件 | 变更 |
|------|------|
| `app.py` | 导航重组；合并热点选题页（+360 行）；移除文案生成页（-520 行）；设置页精简；常量新增；跳转逻辑修改 |
| `generator/copywriter.py` | 新增 `CONTENT_FORMAT_INSTRUCTION` dict；5 个 prompt 模板 + 5 个函数签名均增加 `content_format` 参数 |
| `hotspot/service.py` | `HOTSPOT_COPY_PROMPT` 模板 + `generate_hotspot_copy()` 增加 `content_format` 参数 |
| `scheduler.py` | `_auto_generate()` 读取 `auto_content_format` 并传递 |

## 影响范围

- **破坏性变更**：无。所有旧 session state key、DB setting key、函数签名均向后兼容（新增参数有默认值 `""`）
- **UI 变更**：导航项从 8 个减为 7 个；热点选题页内容显著增加；设置页精简
- **用户体验**：创作流程从「多个页面跳转」变为「单页面切换路径 + 调节参数」
