# PRD：文案向导（CopyWizard）

> 版本：v0.1 | 日期：2026-05-17 | 状态：草稿

---

## 1. 定位

CopyWizard 是 CopyAgent 的**核心创作模块**。它不要求用户直接写 Prompt，而是通过 4 步对话式引导，把产品信息、卖点、用途、风格拆成选择题和填空，最后自动拼装成一条高质量口播文案。

**与旧版的区别：**
- 旧版：一个输入框输入主题 → 自由生成（适合有经验的人）
- 新版：4 步向导，一步一步引导（降低使用门槛，确保输出质量可控）

## 3. 核心功能拆解

### 3.1 产品输入（第 1 步）

| 字段 | 必填 | 说明 |
|------|------|------|
| 产品名称 | ✅ | 你要推广的产品/课程/服务，如 "AI写作训练营" |

- 系统自动记录历史输入的产品名称，下次输入时提供下拉建议
- 如果输入的产品名称和知识库中的课程文档高度相关，后续自动关联

### 3.2 卖点获取（第 2 步）— 双路径

#### 路径 A：用户手动输入

- 提供 3 个空输入框 + "添加更多" 按钮
- 最少填 1 个，最多不限
- 输入框 placeholder 示例："7天写出100条爆款文案"

#### 路径 B：AI 自动搜索（用户跳过时）

用户勾选"帮我自动搜索"后，系统调用 DeepSeek，分两步：

**第一步：搜索产品卖点**

Prompt 示意：
```
请搜索产品"XXX"的核心卖点。输出 5 个卖点，每个一句话，按重要性排序。
如果产品在以下课程知识中有涉及，优先从课程中提取：
{检索到的知识库片段}
```

**第二步：挖掘用户痛点**

Prompt 示意：
```
请分析产品"XXX"的目标用户有哪些核心痛点。
输出 5 个痛点，每个一句话，按强烈程度排序。
```

**输出合并展示：**
```
┌───────────────────────────────────────┐
│ 🔍 自动搜索到以下卖点和痛点，勾选你要用的：       │
│                                        │
│ 卖点:                                  │
│ ☑ 7天掌握AI爆款文案方法论                  │
│ ☑ 每天上班前文案已自动写好                  │
│ ☑ 拆解100+条爆款总结的公式                 │
│ ☐ 不限行业，通用可复制                     │
│ ☐ 课程内容持续更新                        │
│                                        │
│ 痛点:                                  │
│ ☑ 每天没时间写文案                        │
│ ☑ 写出来没有播放量                        │
│ ☐ 不知道怎么写钩子                        │
│ ☑ 灵感不稳定，有时有有时没                  │
│                                        │
│             [确认] [重新搜索]            │
└───────────────────────────────────────┘
```

### 3.3 视频用途（第 3 步）

| 选项 | 文案侧重 | 典型场景 |
|------|---------|---------|
| **卖货/挂车** | 突出产品效果、用户见证、限时优惠、逼单话术 | 小黄车、商品卡 |
| **引流直播间** | 留悬念、引导进直播间、不说完 | 直播预告、引流视频 |
| **种草/品宣** | 建立认知、激发兴趣、不硬卖 | 品牌号日常内容 |
| **通用文案** | 不做特别区分，AI 自由发挥 | 不确定时默认选这个 |

### 3.4 开头风格（第 4 步）

| 风格 | 示例句式 | 适用场景 |
|------|---------|---------|
| **提问式** | "你知道为什么你的文案没流量吗？" | 教育类、干货类 |
| **反问式** | "难道没有天赋就不能写爆款文案？" | 打破偏见、情绪激发 |
| **痛点直击** | "你是不是每天坐到桌前都不知道今天拍什么？" | 强共鸣、快速留人 |
| **数据冲击** | "90% 的口播视频，死在开头前 3 秒。" | 建立权威、引发好奇 |
| **反常识** | "真正能卖货的文案，恰恰不是写得好的那种。" | 颠覆认知、高停留 |
| **故事开头** | "上周我有个学员跟我说，她一个月没出单..." | 代入感、信任感 |
| **悬念** | "接下来这个钩子公式，是我拆了 100 条爆款才总结的。" | 高完播率 |
| **AI 自动选** | 系统根据产品+用途自动匹配最合适的 | 不确定时默认 |

## 4. 生成逻辑

### 4.1 Prompt 组装

```
[系统角色] 你是一位专业的短视频口播文案写手。
           请根据课程知识撰写文案。

[产品信息]  产品：{product}
           卖点：{selling_points}
           痛点：{pain_points}

[视频用途]  {purpose}  →  映射写作策略

[开头风格]  {hook_style}  →  映射开头句式

[课程知识]  {rag_retrieved_chunks}

[输出要求]  口播友好、字数约{length}字、必须有强钩子
           输出: 钩子说明 → 文案正文 → 引用来源
```

### 4.2 用途→策略映射

| 用途 | Prompt 注入 |
|------|-----------|
| 卖货 | "在文案中后段自然地引导下单，强调效果和紧迫感，加入1-2个用户案例" |
| 引流直播间 | "文案制造悬念，结尾不要给出完整答案，引导用户进直播间获取" |
| 种草 | "只做认知教育，不做硬销售，让用户觉得'这个我需要'，但不直接推销" |
| 通用 | 不注入额外策略 |

### 4.3 开头→句式映射

| 开头 | Prompt 注入 |
|------|-----------|
| 提问式 | "开头用一个面向用户的问题切入，让用户产生'对啊，为什么'的思考" |
| 反问式 | "开头用'难道...吗'的反问句式，打破用户的固有认知" |
| 痛点直击 | "开头直接描述用户的一个痛苦场景，用'你是不是也...'的共鸣句式" |
| 数据冲击 | "开头抛出一个具体数据或比例，制造'原来如此'的冲击感" |
| 反常识 | "开头说一个和大众认知相反的结论，然后用正文解释" |
| 故事 | "开头用'前几天/昨天/我有个朋友/学员'引出一个小故事" |
| 悬念 | "开头暗示接下来要说的很重要，用'这个方法/这个公式/这个秘密'等词" |

## 6. 界面布局建议

### 方案 A：独立页面（推荐做 MVP）

```
┌─────────────────────────────────────────────────┐
│  ✍️ 文案向导              [历史记录] [设置]      │
├─────────────────────────────────────────────────┤
│                                                  │
│   ● 第1步：产品  ● 第2步：卖点  ○ 第3步：用途  ○ 第4步：开头  │
│                                                  │
│   ┌─────────────────────────────────────────┐   │
│   │  （当前步骤的内容区域）                      │   │
│   │                                          │   │
│   │                                          │   │
│   │                    [上一步] [下一步]       │   │
│   └─────────────────────────────────────────┘   │
│                                                  │
└─────────────────────────────────────────────────┘
```

上方步骤条显示进度，内容区逐步切换。最后一步完成后进入"生成结果"页。

### 方案 B：单一长表单

所有选项在一个页面，一次性填完点生成。适合熟手快速操作。

**建议两种都做：** 向导模式（默认）+ 快速模式（一键表单）。

## 8. 待确认问题

1. **知识库关联**：第 2 步 AI 自动搜索卖点时，是否要从知识库中检索课程内容来辅助？还是纯靠 DeepSeek 通用搜索？需要
2. **第 4 步默认值**：开头风格默认选"AI 自动选"还是"痛点直击"？除了痛点直击还有其他的短视频爆款开头的写作方式也可以自动选择
3. **多产品**：是否支持一次输入多个产品对比？还是每次只做一个产品？可以做对比，如你还在用这种吵的像拖拉机一样的氧气泵吗？我劝你快试试这款氧气泵...。
4. **生成数量**：一次生成 1 条还是多条（不同开头风格各来一条）3条？
5. **用户点击立即写作后要有一个进度条和点击动画反馈或者弹窗，要有实时的进度，完成后自动点击跳转到生成后的文案中。
