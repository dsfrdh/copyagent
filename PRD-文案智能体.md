# PRD：AI 文案智能体（CopyAgent）

> 版本：v0.1 | 日期：2026-05-17 | 状态：草稿

---

## 1. 项目背景与目标

### 1.1 背景

用户每天需要为短视频创作口播文案，核心工作流有三步：

1. **拆解爆款**：收集同赛道热门视频，分析其结构、钩子、情绪曲线
2. **参考改写**：基于拆解结论，结合自有课程方法论，重写出新的口播文案
3. **定时产出**：在上桌前自动完成当日文案，上班即可拍

当前痛点：全部依赖人工完成，拆解耗时长、灵感不稳定、无法保证每日产出节奏。

### 1.2 目标

构建一个**本地桌面应用**，实现：

- 每日定时自动生成 1~N 条口播文案
- 支持手动输入爆款链接/文本，一键拆解结构化分析
- 基于用户自有课程文档构建知识库，所有生成遵循课程方法论
- 本地运行，数据不外泄

## 2. 用户画像

| 维度 | 描述 |
|------|------|
| 身份 | 短视频内容创作者 / 课程讲师 |
| 核心场景 | 每天上班前需要拿到当日拍摄用的口播文案 |
| 使用频率 | 每日自动运行 + 不定时手动拆解/改写 |
| 技术能力 | 非开发背景，能使用桌面软件 |
| 设备 | Windows 台式机（主力），可能扩展到 Mac |
| 隐私要求 | 高 — 课程内容、文案策略均不能上传到云端 |

## 3. 功能需求

### 3.1 知识库管理（P0）

**3.1.1 课程文档导入**
- 支持导入 Markdown（.md）、Word（.docx）、PDF（.pdf）、纯文本（.txt）
- 导入时自动解析、分块、向量化存入本地知识库
- 支持文件夹批量导入、拖拽导入

**3.1.2 知识库管理**
- 查看已导入文档列表，支持删除、重新解析
- 查看分块状态（文档数、总段落数、向量化状态）
- 支持按标签/主题分组管理（如「开头钩子」「信任构建」「成交话术」）

**3.1.3 课程知识点标注**
- 用户可对特定段落打标签：「钩子模板」「金句」「痛点描述」「产品卖点」「成交话术」
- 标签参与后续生成时的检索权重

### 3.2 爆款拆解（P0）

**3.2.1 输入方式**
- **粘贴文本**：直接粘贴爆款视频的字幕/文案
- **URL 解析**：粘贴短视频链接（抖音/小红书/B站），自动抓取文案（如有）
- **批量导入**：一次粘贴多条文案，逐条拆解

**3.2.2 拆解输出**
- **结构拆解**：黄金圈/SCQA/AIDA/钩子-展开-高潮-收尾 等模型自动识别
- **情绪曲线**：逐句标注情绪标签（好奇/震惊/共鸣/焦虑/信任/行动）
- **钩子分析**：识别开头的钩子类型（反常识/数据/悬念/痛点直击/故事开头）
- **金句提取**：自动标出可复用的句式/表达
- **节奏分析**：短句/长句分布、段落切换频率
- **关键词提取**：高频词、话题标签建议

**3.2.3 拆解结果存储**
- 每次拆解结果存入「爆款库」
- 支持按标签、日期、来源检索
- 生成文案时可引用某条爆款的结构或风格

### 3.3 文案生成（P0）

**3.3.1 生成模式**

| 模式 | 说明 |
|------|------|
| **自由创作** | 输入主题/关键词，基于课程知识库自由生成 |
| **仿写** | 选择一条爆款拆解记录，保留其结构，用课程内容重新填充 |
| **改写润色** | 已有草稿，让 AI 按课程方法论优化 |
| **组合生成** | 钩子用爆款A的结构 + 展开用爆款B的逻辑 + 收尾用自己的成交话术 |

**3.3.2 生成参数**
- **文案长度**：30秒 / 60秒 / 90秒 / 自定义字数
- **风格**：口语化 / 正式 / 幽默 / 情绪化 / 专业感
- **目的**：涨粉 / 引流 / 成交 / 种草 / 品牌
- **生成数量**：1~5 条可选
- **口播感**：自动优化为适合念出来的口语节奏

**3.3.3 生成依据**
- 优先从知识库检索相关课程内容作为素材
- 若用户指定了爆款拆解记录，则约束结构模型
- 每次生成附带「引用来源」——标注使用了课程的哪个知识点

### 3.4 定时自动化（P1）

**3.4.1 定时任务配置**
- 设置每日生成时间（如每天 7:00）
- 设置每日生成数量（默认 3 条）
- 设置默认主题方向（可每天不同：周一涨粉/周二成交/...）
- 工作日/每天两种模式

**3.4.2 生成结果呈现**
- 在应用首页按日期展示当日生成的文案列表
- 每条文案可预览、打分、编辑、导出
- 未查看的文案有明显标记（红点/未读状态）

**3.4.3 任务执行**
- 应用支持开机自启、后台常驻
- 定时任务在后台静默执行，完成后系统通知
- 支持手动触发「立即生成今日份」

### 3.5 文案管理（P1）

**3.5.1 历史记录**
- 按日历视图浏览历史生成记录
- 支持搜索（关键词、日期范围、标签）
- 标记状态：待拍 / 已拍 / 数据好 / 数据一般 / 弃用

**3.5.2 版本对比**
- 改写模式保留改写前后对比
- 支持回退到之前的版本

**3.5.3 导出**
- 复制到剪贴板（一键）
- 导出为 Markdown / TXT / 飞书文档
- 导出为提词器格式（每行显示一句，大字号）

### 3.6 数据反馈闭环（P2）

**3.6.1 数据录入**
- 每条文案拍摄发布后可录入表现数据：播放量/点赞/评论/转发/转化数
- 简单评级：爆款/正常/扑街

**3.6.2 效果分析**
- 按钩子类型/结构模型/风格 归因分析
- 「你的账号里，哪种钩子最有效？」
- 反馈数据影响后续生成的偏好权重

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 响应时间 | 单条文案生成 < 30s，拆解分析 < 15s |
| 离线运行 | 核心功能（生成、拆解）完全离线，不依赖外部 API |
| 数据安全 | 所有数据（知识库、文案、拆解记录）仅存储在本地 |
| 可扩展性 | 支持后期接入云端模型（可选），作为本地模型的补充 |
| 兼容性 | Windows 10/11 64位，后续扩展 macOS |
| 资源占用 | 后台常驻内存 < 500MB，生成时峰值 < 4GB |
| 容错 | 定时任务失败自动重试，失败通知用户 |

### 技术选型

| 层 | 技术 | 理由 |
|----|------|------|
| 桌面框架 | Electron + React | 跨平台，生态丰富，文件系统访问友好 |
| 本地 LLM | Ollama (Qwen 3 / DeepSeek) | 本地推理，中文能力强 |
| 向量数据库 | LanceDB | 嵌入式，无需单独服务，支持增量写入 |
| 嵌入模型 | BGE-M3 或 text2vec-large-chinese | 中文语义检索效果好 |
| 元数据存储 | SQLite | 轻量，单文件，零配置 |
| 任务调度 | node-cron | 定时任务，简单可靠 |
| 文档解析 | 自研解析器 (markdown-it + mammoth + pdf-parse) | 多格式支持 |

## 6. 核心数据模型

```
KnowledgeDoc         知识库文档
  id, title, file_path, file_type, tags, created_at, updated_at
  chunks: [KnowledgeChunk]

KnowledgeChunk       知识块
  id, doc_id, content, chunk_index, tags, embedding_vector, created_at

ViralAnalysis         爆款拆解记录
  id, source_url, raw_text, title, tags
  structure_type      枚举: scqa | aida | hook-expand-climax | ...
  hook_type           枚举: counterintuitive | data | suspense | pain_point | story
  emotion_curve       JSON: [{sentence_idx, emotion_label, intensity}]
  golden_sentences    JSON: [string]
  analysis_text       完整分析报告
  created_at

CopyTask              文案生成任务
  id, mode            枚举: free | rewrite | imitate | combine
  target_date         目标日期（定时任务使用）
  params              JSON: {length, style, purpose, count, topic, ...}
  status              枚举: pending | running | completed | failed
  created_at, completed_at

Copy                  生成的文案
  id, task_id, title, content, version
  source_chunks       JSON: [{chunk_id, relevance_score}]
  reference_analysis  关联的爆款拆解ID（可为空）
  rating              用户评分 1-5
  publish_status      枚举: pending | shot | published
  publish_data        JSON: {views, likes, comments, shares, conversions}
  created_at, updated_at
```

## 7. 交互流程

### 7.1 每日自动化流程

```
前一天 23:59 ─→ 系统检查明日任务配置
     │
每天 7:00  ─→ 触发定时任务
     │         ├─ 读取知识库
     │         ├─ 可选：参考近期高评分爆款结构
     │         ├─ 调用 LLM 生成 3 条文案
     │         ├─ 存入本地数据库
     │         └─ 系统通知："今日文案已生成"
     │
用户 9:00 ─→ 打开应用
     │         ├─ 首页展示今日３条文案
     │         ├─ 逐条预览 / 编辑 / 打分
     │         └─ 复制文案 → 去拍摄
```

### 7.2 爆款拆解 → 仿写流程

```
用户粘贴爆款文本
     │
     ▼
点击「拆解」
     │
     ├─ LLM 分析：结构模型 / 钩子类型 / 情绪曲线 / 金句
     │
     ▼
拆解结果展示（可视化卡片 + 情绪曲线图）
     │
     ▼
点击「仿写」
     │
     ├─ 选择参考维度：□结构 □钩子 □情绪节奏 □金句风格
     ├─ 输入主题（或留空自动匹配课程内容）
     │
     ▼
生成 2~3 个版本，并排对比
     │
     ▼
选择版本 → 微调 → 复制使用
```

## 9. 风险与假设

| 风险 | 影响 | 缓解策略 |
|------|------|----------|
| 本地 LLM 中文生成质量不达标 | 文案不可用 | 前期多模型对比测试；支持切换云端 API 作为备选 |
| 本地模型推理速度慢 | 用户体验差 | 选择 7B-14B 参数模型，量化版本；GPU 加速 |
| 向量检索不准 | 生成内容偏离课程体系 | 调优 chunk 粒度；支持用户手动标注关键段落提权 |
| OCR 后续版本支持的文档格式解析失败 | 知识库不完整 | 先支持纯文本 + Markdown，逐步扩展 |
| 开机自启在不同 Windows 版本表现不一 | 定时任务不可靠 | 提供手动「立即生成」作为兜底 |

### 假设

- 用户有一块 ≥ 8GB 显存的 NVIDIA GPU（用于本地模型推理加速），或至少 16GB 内存（CPU 推理）
- 课程文档以结构化文本为主，非扫描件
- 用户有基本的软件安装能力（装 Ollama、装应用本身）
- 每天生成的文案经过人工筛选后再使用，AI 是辅助而非替代

## 10. 附录：关键 Prompt 模板参考

### 拆解 Prompt（示意）

```
你是一位资深的短视频文案分析师。请分析以下口播文案：

【待分析文本】
{user_input}

请从以下维度输出结构化分析：
1. 结构模型：识别使用了哪种说服结构（SCQA/AIDA/黄金圈/钩子-展开-高潮-收尾/其他）
2. 钩子分析：开头的钩子类型、为什么有效、可复用的句式
3. 情绪曲线：逐句标注情绪类型和强度（1-5），用 JSON 输出
4. 金句提取：列出可直接复用或改写的高价值表达
5. 节奏特点：句长分布、转折频率、口语化程度
```

### 生成 Prompt（示意）

```
你是一位{课程领域}的短视频文案写手。请基于以下课程知识，撰写一条{length}的口播文案。

【课程参考】
{retrieved_chunks}

【风格要求】
- 目的：{purpose}
- 风格：{style}
- 必须口播友好，适合照着念

{仿写约束}
{additional_constraints_if_imitate_mode}

输出要求：
1. 文案正文
2. 使用的课程知识点列表
3. 钩子类型说明
```
