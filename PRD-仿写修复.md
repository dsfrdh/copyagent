## 1. 问题

当前「仿写爆款」的 Prompt 写死了"内容完全换成课程相关的"，导致：

- 用户拿一条**鱼缸氧气泵**的爆款做结构模板
- AI 把内容换成了课程知识库里的主题
- 用户想要的是：**保留氧气泵的结构，写加热棒的文案**

## 3. Prompt 修改

### 旧 Prompt（有问题）

```
保留原爆款的钩子类型、情绪推进节奏、段落结构
但内容完全换成课程相关的，不要抄袭原句
```

### 新 Prompt

```
你要写文案的产品是：{product}
该产品的卖点：{selling_points}
该产品的用户痛点：{pain_points}

请模仿下面这条爆款文案的【结构和节奏】，为「{product}」撰写一条新文案。

【爆款原文（供参考结构和风格）】
{original_text}

【爆款结构拆解】
{structure_analysis}

【课程参考资料】
{knowledge}

写作要求：
- 产品是「{product}」，内容必须围绕这个产品展开
- 保留爆款原文的：钩子类型、段落推进节奏、句式长短结构
- 替换内容为「{product}」的相关信息和卖点
- 不要直接抄袭爆款原文的句子
- 用课程方法论来组织说服逻辑
- 字数：约{max_words}字，{length}长度
- 风格：{style}
```

**关键变化：**
1. 爆款原文作为参考附上（不只是结构拆解）
2. 产品名、卖点、痛点作为必填参数传入
3. 指令从"换成课程内容" → "为 {product} 写文案"
4. 结构参考 + 内容分离，职责清晰

## 4. 与向导模式的关系

| 模式 | 产品来源 | 结构来源 | 卖点来源 |
|------|---------|---------|---------|
| **向导模式** | 用户输入 | 无模板，AI 自由组织 | 手动填 or AI 搜 |
| **仿写模式** | 用户输入 | 爆款拆解记录的结构 | 手动填 or AI 搜 |

**两者的 Step 2（卖点搜索）可以复用同一套逻辑**——就是已经写好的 `search_selling_points()` 函数。

因此实际只需要：
1. 改 `IMITATE_PROMPT` → 加入 product/selling_points/pain_points 参数
2. 改 `generate_imitate()` → 接收新参数，调用 `search_selling_points()`
3. 改 app.py 仿写 UI → 在选爆款后加产品输入 + 卖点搜索步骤
